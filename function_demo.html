1.1 姓名非空验证

验证机制说明:
  • 在 addPerson() 方法中，使用 while 循环持续提示用户输入，直到输入非空姓名
  • 验证条件: !name.empty()
  • 如果输入为空，显示错误提示并要求重新输入

代码实现参考:

string name;

三、容量限制功能演示

3.1 容量限制机制说明

容量限制:
  • 通讯录最大容量为1000人（定义在宏 Max 中）
  • 在 addPerson() 方法中，首先检查通讯录是否已满
  • 如果已满，显示"通讯录已满"提示并返回，不执行添加操作

代码实现参考:

#define Max 1000

void addPerson()
{
    if (isFull())
    {
        cout << "通讯录已满" << endl;
        return;
    }
    // ... 添加联系人逻辑
}

bool isFull() const { return Size == Max; }

3.4 容量限制用户通知

系统行为:
  • 当通讯录已满时，系统仅输出一行提示信息："通讯录已满"
  • 用户无法继续添加新联系人
  • 用户可以执行其他操作（显示、查找、修改、删除、清空、搜索）
  • 删除联系人后，通讯录容量释放，可以重新添加
通知方式:
  • 文本提示："通讯录已满"
  • 无额外警告或弹窗
  • 操作简单直接，用户一目了然
